Spring 시작하기

개요

  • 스프링 프레임워크는 자바 언어 기반의 웹 프레임워크이다.
  • 개발군 중 백엔드(Backend)에 속하는 프레임워크로, 서버 개발을 담당한다.
  • 스프링을 배우는 이유 중 하나로 한국 전자정부표준프레임워크의 기반 기술이며 한국정보화진흥원에서는 공공기관의 웹 서비스 제공 시 스프링을 권장하고 있다.

스프링의 특징

특징 설명
POJO(Plain Old Java Object) 별도의 프레임워크 없이 Java EE를 사용할 때에 비해 특정 인터페이스를 직접 구현하거나 상속받을 필요가 없어 기존 라이브러리를 지원하기가 용이하고, 객체가 가볍다.
관점 지향 프로그래밍(Aspect Oriented Programming, AOP) 로깅, 트랜잭션, 보안 등 여러 모듈에서 공통적으로 사용하는 기능을 분리하여 관리할 수 있다.
의존성 주입(Dependency Injection, DI) 프로그래밍에서 구성요소 간의 의존 관계가 소스코드 내부가 아닌 외부에서 설정을 통해 정의되는 방식이다. 코드 재사용을 높여 소스코드를 다양한 곳에 사용할 수 있으며 모듈간의 결합도도 낮출 수 있다. 계층, 서비스 간에 의존성이 존재하는 경우 스프링 프레임워크가 서로 연결시켜준다.
제어 역전(Inversion of Control, IoC) 제어 역전은 이와 반대로 외부 라이브러리 코드가 개발자의 코드를 호출하게 된다. 즉, 제어권이 프레임워크에게 있어 필요에 따라 스프링 프레임워크가 사용자의 코드를 호출한다.
생명주기 관리 스프링 프레임워크는 Java 객체의 생성, 소멸을 직접 관리하며 필요한 객체만 사용할 수 있다.

프로젝트 생성하기

개발환경

  • Java 설치
  • IDE : IntelliJ (Ultimate 버젼이 Spring을 지원해주지만, Community 버젼 기준으로 설명, 돈 없음..ㅠ)

생성하기

  1. https://start.spring.io 에서, 스프링 프로젝트를 만들 때 빌드방식(Gradle, Maven)이나 프로그래밍 언어(Java, Kotlin, Groovy)를 정하고 스프링 버젼 및 필요한 Dependencies를 지정할 수 있다.

  2. 저는 위와 같이 설정했습니다. Spring은 백엔드 개발로 프론트엔드 개발을 따로 하실거면, Dependencies에서 Thymeleaf는 빼셔도 됩니다.

  3. GENERATE를 누르면, 압축파일을 다운받게 됩니다. 원하는 위치에 두시고 압축을 푼 후, IntelliJ로 build.gradle을 열어줍니다. [Open as Project]로 선택해서 열어주시면 됩니다.

  4. 처음 실행하게 되면, 필요한 파일들을 다운받는 과정으로 시간이 필요합니다. 여기까지 진행하셨으면, 프로젝트 생성은 끝입니다!
